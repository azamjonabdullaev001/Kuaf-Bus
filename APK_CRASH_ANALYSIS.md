## ❌ НАЙДЕННЫЕ КРИТИЧЕСКИЕ ПРОБЛЕМЫ:

### **1️⃣ ОТСУТСТВУЕТ GOOGLE MAPS API КЛЮЧ** (КРИТИЧНО!)

**Проблема:**
- `react-native-maps` требует Google Maps API ключ для Android
- В `app.json` не было секции `android.config.googleMaps.apiKey`
- БЕЗ ключа приложение **вылетает** при попытке загрузить карту

**Решение:**
- ✅ Добавлена секция в `app.json`:
```json
"android": {
  "config": {
    "googleMaps": {
      "apiKey": "AIzaSyDummy_Key_Replace_This"
    }
  }
}
```
- 📖 Создана инструкция: `GOOGLE_MAPS_API_KEY_GUIDE.md`
- ⚠️ **НУЖНО ЗАМЕНИТЬ** на реальный ключ из Google Cloud Console

---

### **2️⃣ СЛОЖНЫЙ КОМПОНЕНТ BUSMARKER**

**Проблема:**
- Вложенный `<View>` компонент `BusMarker` внутри `<Marker>`
- В production APK это вызывает нестабильность
- React Native Maps плохо работает с кастомными вложенными компонентами

**Решение:**
- ✅ Убран компонент `BusMarker` из `YandexMap.js`
- ✅ Используются стандартные маркеры с `pinColor`
- ✅ Удалены неиспользуемые styles (busMarker, busBody, busWindow, и т.д.)

---

### **3️⃣ ОТСУТСТВИЕ ERRORBOUNDARY**

**Проблема:**
- Любая ошибка в React компоненте вызывает полный вылет приложения
- Нет обработки неожиданных ошибок

**Решение:**
- ✅ Создан компонент `ErrorBoundary.js`
- ✅ Обернуто всё приложение в `<ErrorBoundary>` в `App.js`
- ✅ Теперь при ошибке показывается экран с кнопкой "Попробовать снова"

---

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### **Почему react-native-maps требует API ключ?**

React Native Maps использует **Google Maps SDK для Android**.  
Без API ключа:
- Android не может загрузить тайлы карты
- Приложение получает ошибку от Google
- React Native ломается и вылетает

### **Почему не работал BusMarker?**

В Expo/React Native **НЕЛЬЗЯ** вкладывать сложные компоненты с `View` внутрь `<Marker>`.  
Это работает в Expo Go (development), НО ломается в production APK.

### **Почему добавили ErrorBoundary?**

Без ErrorBoundary любая ошибка в React = **полный вылет APK**.  
С ErrorBoundary = пользователь видит экран ошибки и может попробовать снова.
