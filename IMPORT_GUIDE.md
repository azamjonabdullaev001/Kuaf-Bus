# Руководство по импорту студентов и водителей

## Поддерживаемые форматы

Система поддерживает импорт из следующих форматов файлов:
- **Excel** (.xlsx, .xls)
- **JSON** (.json)
- **TXT/CSV** (.txt, .csv)

## Новинка: Автоматическое сопоставление колонок ✨

Система **автоматически определяет** какие колонки из вашего файла соответствуют полям базы данных, даже если они названы по-другому!

**Вам не нужно переименовывать колонки** - просто загрузите файл и проверьте автоматическое сопоставление. Если система ошиблась, вы можете вручную выбрать правильные колонки.

## Обязательные поля

Каждый файл должен содержать следующие поля (названия могут быть любыми):

| Поле | Описание | Обязательно |
|------|----------|-------------|
| Университетский ID | Логин пользователя | ✓ |
| Имя | Имя | ✓ |
| Фамилия | Фамилия | ✓ |
| Отчество | Отчество | ✗ |

## Форматы файлов

### 1. Excel (.xlsx)

Скачайте шаблон через кнопку "📥 Шаблон" в админ-панели или создайте файл со следующей структурой:

| university_id | first_name | last_name | middle_name |
|--------------|------------|-----------|-------------|
| 12345678 | Иван | Иванов | Иванович |
| 87654321 | Мария | Петрова | Сергеевна |
| 11223344 | Алексей | Сидоров | Дмитриевич |

### 2. JSON (.json)

```json
[
  {
    "university_id": "12345678",
    "first_name": "Иван",
    "last_name": "Иванов",
    "middle_name": "Иванович"
  },
  {
    "university_id": "87654321",
    "first_name": "Мария",
    "last_name": "Петрова",
    "middle_name": "Сергеевна"
  }
]
```

### 3. TXT/CSV (.txt, .csv)

**Разделитель:** табуляция или запятая

```
university_id	first_name	last_name	middle_name
12345678	Иван	Иванов	Иванович
87654321	Мария	Петрова	Сергеевна
11223344	Алексей	Сидоров	Дмитриевич
```

или

```
university_id,first_name,last_name,middle_name
12345678,Иван,Иванов,Иванович
87654321,Мария,Петрова,Сергеевна
11223344,Алексей,Сидоров,Дмитриевич
```

## Альтернативные названия колонок

Система автоматически распознает следующие варианты названий:

| Поле | Варианты |
|------|----------|
| university_id | university_id, Университетский ID, id |
| first_name | first_name, Имя, name |
| last_name | last_name, Фамилия, surname |
| middle_name | middle_name, Отчество, patronymic |

## Как использовать

1. Откройте админ-панель: http://localhost:3001
2. Войдите с правами администратора
3. Выберите вкладку "Студенты" или "Водители"
4. Нажмите кнопку "📁 Импорт"
5. Выберите файл в одном из поддерживаемых форматов
6. **Проверьте автоматическое сопоставление колонок**
   - Система попытается автоматически определить соответствие
   - Если нужно, вручную выберите правильные колонки из выпадающих списков
   - Обязательные поля отмечены звездочкой (*)
   - Для каждого поля показан пример данных из вашего файла
7. Нажмите "Далее →"
8. Проверьте предв**детальный отчет**:
  - Количество успешно импортированных записей
  - Количество ошибок
  - Список первых 5 ошибок с деталями
- Дубликаты university_id будут отклонены (уникальное поле)
- После импорта можно повторить попытку с исправленными данными

## Примеры названий колонок

Система **автоматически распознает** следующие варианты:

### Университетский ID
- `university_id`, `Student_ID`, `Driver_ID`
- `id`, `ID`, `№`
- `Университетский ID`, `ID студента`

### Имя
- `first_name`, `Name`, `FirstName`
- `name`, `имя`, `Имя`

### Фамилия
- `last_name`, `Surname`, `LastName`
- `surname`, `фамилия`, `Фамилия`

### Отчество
- `middle_name`, `Father_Name`, `Patronymic`
- `patronymic`, `отчество`, `Отчество`

**Пример:** Даже если ваш файл использует колонки `Student_ID`, `Name`, `Surname`, `Father_Name` - система автоматически их распознает!

## Обработка ошибок

- Если некоторые записи содержат ошибки, они будут пропущены
- Система покажет количество успешно импортированных и неудачных записей
- Дубликаты university_id будут отклонены (уникальное поле)
 (стандартные колонки)
- `import-example.txt` - пример TXT файла (стандартные колонки)
- `import-custom-columns.txt` - пример с **нестандартными названиями колонок** (Student_ID, Name, Surname, Father_Name)

## Тестирование функции маппинга

Чтобы проверить автоматическое сопоставление колонок:
1. Используйте файл `import-custom-columns.txt`
2. При импорте система автоматически определит:
   - `Student_ID` → Университетский ID
   - `Name` → Имя
   - `Surname` → Фамилия
   - `Father_Name` → Отчество
3. Вы можете изменить сопоставление вручную, если потребуется

В корне проекта находятся примеры:
- `import-example.json` - пример JSON файла
- `import-example.txt` - пример TXT файла

## Установка зависимостей

Для работы импорта необходимо установить зависимости админ-панели:

```bash
cd admin-panel
npm install
```

## Пересборка контейнера

После установки зависимостей пересоберите контейнер:

```bash
docker-compose build admin-panel
docker-compose up -d admin-panel
```
